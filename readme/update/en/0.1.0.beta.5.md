## 0.1.0.beta.5 Update (26.06.07)


### Added `api()`

You can now set the API path later in the builder chain.


#### Function Signature

```ts
    /**
     * Sets the API URL.
     */
    api: <P = T>(api: string) => R,
```

As shown in the function signature, you can set or change the API path during the chain and change the existing `T` type to `P`.

---
### Changed the `dench` Factory Function Signature

- You no longer need to provide an API path when creating a `dench` request builder.
- `post` and `put` also no longer require data when the builder is created.

#### Function Signature

```ts
    export interface DenchInterface{
        baseURL : DenchHTTPURL,
        get : <T>(api?:string) => DenchGetBuilder<T>
        post : <T>(api?: string, data?: unknown) => DenchCreateBuilder<T>
        put : <T>(api?: string, data?: unknown) => DenchCreateBuilder<T>
        delete : <T>(api?: string) => DenchGetBuilder<T>
    }
```

---
### Made Data Arguments Optional

- You can now provide data directly to `sendJson`, `sendForm`, and `sendBlob`.

#### Function Signature

```ts
    sendJson : (data? : unknown) => DenchCreateBuilder<T>,
    sendForm : (data? : FormData) => DenchCreateBuilder<T>,
    sendBlob : (data? : Blob) => DenchCreateBuilder<T>,
```

If data was already provided when creating a `dench.put` or `dench.post` builder, it will be overwritten by the data passed to these methods.

---
### Added `sendRaw`, `sendUrlEncoded`, and `DenchURLSearchParams`

- You can now use `sendUrlEncoded` to send URL-encoded parameters and `sendRaw` to send unmodified raw data.

```ts
    type DenchURLSearchParams = ConstructorParameters<typeof URLSearchParams>[0]

    /**
     * @header { "Content-Type" : "application/x-www-form-urlencoded" }
     */

    sendUrlEncoded? : (data?  : DenchURLSearchParams) => DenchCreateBuilder<T>

    /**
     * @header { "Content-Type" : "application/octet-stream" }
     */
    sendRaw? : (data? : BodyInit) => DenchCreateBuilder<T>
```

`DenchURLSearchParams` strictly defines the accepted argument types for `sendUrlEncoded`. It is equivalent to the constructor parameter type of `URLSearchParams`:

```ts
type DenchURLSearchParams = string | string[][] | Record<string, string> | URLSearchParams | undefined
```

Use `sendRaw` when you want to process data yourself and place it directly into the request body.
For data that has already been processed with operations such as `JSON.stringify()` or `FormData()`, use `sendRaw`.

---

### `copy()` Method

You can now copy a builder with `copy()`.

This feature was added to improve builder reusability. The following example demonstrates how it works:

```ts
const denchInstance = dench('https://jsonplaceholder.typicode.com', "testInstance");


const sampleBuilder = denchInstance.post()
                .abort(new AbortController())
                .timeout(5000)
                .error((error) => console.error('Error occurred:', error))

const builder1 = sampleBuilder.copy()
.sendJson({
    userId : 1,
    title : 'delectus aut autem',
    completed : false
})
.api('/todos?userId=1');

const builder2 = sampleBuilder.copy()
.sendForm(new FormData())
.api('/todos?userId=1');
```

Previously, reusing common settings such as `error` and `timeout` required copying the builder through destructuring or creating a new builder. With `copy`, you can copy an existing builder configuration and reuse it as shown above.
