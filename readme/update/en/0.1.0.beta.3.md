## 0.1.0.beta.3 Update (26.06.06)


### DenchURL

You can now use the `DenchURL` type. `DenchURL` is defined as follows:

```tsx
export type DenchHTTPURL =
`http://${string}`
| `https://${string}`
| `file://${string}`
| `ftp://${string}`
| `ws://${string}`
| `wss://${string}`
```

This type was introduced to reduce URL typing mistakes.

It has also been applied to the `dench` function:

```ts
export function dench<T>(baseURL: DenchHTTPURL, label? :string) : DenchInterface
```

**However, some developers may want to use a less restrictive format for `baseURL`, so there are plans to provide multiple, more specific variants of the `dench` function.**

### Updated `auth` Function and `DenchAuthType`

The `auth` function and `DenchAuthType` have been updated. The `auth` function now has the following signature:

```ts
    auth: (token: string, type? : DenchAuthType) => R,

    // Previous version
    auth: (token: string) => R,
```

Previously, `auth` only supported Bearer authentication. **You can now specify a `DenchAuthType` to authenticate tokens using your preferred scheme.** The following authentication schemes are supported:

```ts
export enum DenchAuthType{
    ETC= 'ETC',
    BASIC = 'Basic',
    BEARER = 'Bearer',
    DIGEST = 'Digest',
    NEGOTIATE = 'Negotiate',
    AWS4_HMAC_SHA256 = 'AWS4-HMAC-SHA256',
    API_KEY = 'ApiKey',
    TOKEN = 'Token',
    HMAC = 'HMAC',
}
```

If `type` is omitted, the default value is **`DenchAuthType.BEARER`**.

For any other authentication scheme, use `DenchAuthType.ETC` and provide the complete authentication value as the `token` **full string**.
